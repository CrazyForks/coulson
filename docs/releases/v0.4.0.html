Coulson 0.4.0

March 24, 2026
New
  • Docker Compose provider — auto-detect and manage Compose apps with idle reaping process
  • Launchd socket activation replaces pf for port 80/443 forwarding forward
  • Hook mechanism for app lifecycle events (app_add, app_ready, app_stop, etc.) hooks
  • Per-app hooks via [hooks] section in .coulson.toml with shell + webhook support hooks
  • Tunnel lifecycle hooks: tunnel_start / tunnel_stop hooks
  • Env injection via [env], env_url, and env_url_headers in .coulson.toml process
  • Per-app cname alias for custom domain routing proxy
  • CLI coulson open command to open app URL in browser cli
  • Block non-browser clients until managed process is ready (instead of 502) proxy
  • mDNS periodic health check to detect stale registrations mdns
  • Linux x86_64 build in CI ci
Fixed
  • Dual-stack bind panic — use [::] instead of 0.0.0.0 proxy
  • Graceful shutdown shortened from 300s to 3s+5s proxy
  • Node provider defers to Procfile when web process exists process
  • Dashboard domain links now include port when forward target doesn't match dashboard
  • Rebuild mDNS daemon on network change (Wi-Fi switch, sleep/wake) mdns
  • Tunnel domain validation rejects wildcards, IPs, and malformed input control
  • CLI and tunnel commands made idempotent for provisioning safety cli
  • Cross-platform build fix for non-macOS forward stubs build
Changed
  • Unified app configuration to .coulson.toml — legacy formats deprecated config
  • Dropped granian support — uvicorn is now the only ASGI server process
  • X-Forwarded-Host header added to upstream requests proxy
  • Hook context factory extracted — cleaner internal architecture refactor
  • Process manager env injection consolidated — manifest loaded once per start refactor
